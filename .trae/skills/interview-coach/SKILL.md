# 何时使用

当用户出现以下情况时调用此技能：

- 用户提到面试、面试问题、面试总结
- 用户需要记录面试信息
- 用户询问面试技巧或面试准备
- 用户要求查看或更新面试记录
- 用户需要目标公司行业分析
- 用户需要职业机会识别与把握建议

**注意**：如果用户明确说"在面试现场"、"给面试官演示"、"正在面试"，请调用 interview-demo skill，而不是本 skill。

# 核心能力

## 能力一：面试记录管理

### 面试记录文件用途

面试记录文件用于记录每次面试的详细信息，包括公司背景、岗位要求、面试准备、面试过程。该文件会不断追加新的面试记录，用于总结经验、优化面试策略。

### 文件格式规范

每次面试记录应包含以下结构：

```markdown
## [公司名称] [岗位名称]

### 基本信息
- **公司名称**：xxx
- **面试岗位**：xxx
- **面试时间**：xxxx年xx月xx日xx点
- **面试官**：xxx

### 公司背景
- **公司简介**：xxx
- **业务范围**：xxx
- **战略方向**：xxx

### 岗位要求
- **岗位职责**：xxx
- **任职要求**：xxx

### 面试准备
- **能力差距分析**：xxx
- **管理经验重构**：xxx
- **探索型团队考核方案**：xxx

### 面试记录
- **面试时间**：xxx
- **面试官**：xxx
- **面试问题**：xxx
- **面试反馈**：xxx
- **后续安排**：xxx
```

### 如何追加新的面试记录

1. **在文件末尾添加新的二级标题**：

   ```markdown
   ## [公司名称] [岗位名称]
   ```
2. **按上述格式填写完整信息**：

   - 基本信息（公司名称、面试岗位、面试时间、面试官）
   - 公司背景（公司简介、业务范围、战略方向）
   - 岗位要求（岗位职责、任职要求）
   - 面试准备（能力差距分析、管理经验重构、探索型团队考核方案等）
   - 面试记录（面试时间、面试官、面试问题、面试反馈、后续安排）
3. **更新文件末尾的最后更新时间**：

   ```markdown
   ---
   *最后更新：2026-03-25*
   ```

### 注意事项

- 如实记录，不添加假设内容，无需你给出任何建议。

# 工作流程

## 第一步：判断用户需求

根据用户输入判断当前需求：

| 需求类型 | 判断依据 |
|----------|----------|
| **记录需求** | 用户要记录面试信息、查看历史面试 |
| **指导需求** | 用户询问面试技巧、准备建议、行业分析 |

## 第二步：执行对应流程

### 记录模式流程
1. **查看现有面试记录** - 读取 `knowledge/interview.md` 了解历史面试情况
2. **记录到文件** - 按照规范格式追加到 `knowledge/interview.md`
3. **提供分析和建议** - 基于面试记录提供改进建议

### 指导模式流程
1. **了解用户背景** - 读取 `knowledge/resume&projects.md` 和 `knowledge/career-strategy.md`
2. **分析问题** - 结合用户背景和当前求职情况
3. **给出建议** - 提供针对性的面试策略和技巧
