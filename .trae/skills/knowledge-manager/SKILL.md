# Knowledge Manager

## 职责

管理 `knowledge/` 目录下的知识库文件，执行增删改查操作，并在每次操作后自动同步更新 `AGENTS.md` 文件中的相关内容。

## 核心原则

1. **优先复用现有文件** - 默认情况下，新增内容时先查阅 AGENTS.md 中的"知识库索引"表，找到对应角度的现有文件进行更新，不新建文件
2. **按需新建文件** - 仅当用户明确要求创建新文件，或现有文件确实无法容纳新内容的角度时，才新建文件
3. **按角度分类记录** - 根据内容的角度，将信息记录到对应的知识库文件
4. **支持多文件更新** - 一次对话涉及多个角度时，分别更新到不同文件

## 知识库文件规范

### 文件位置

所有知识库文件必须存放在 `knowledge/` 目录下。

### 文件头格式（YAML Front Matter）

每个知识库文件必须包含统一的 YAML 文件头：

```yaml
---
title: "文件标题"
tags: ["标签1", "标签2", "标签3"]
category: "分类名称"
description: "文件内容简要描述"
last_updated: "YYYY-MM-DD"
---
```

#### 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `title` | 是 | 文件标题，简洁明了 |
| `tags` | 是 | 关键词标签，3-8个，用于检索和分类 |
| `category` | 是 | 所属分类，如：职业发展、健康管理、个人成长等 |
| `description` | 是 | 内容摘要，50字以内 |
| `last_updated` | 是 | 最后更新日期，格式：YYYY-MM-DD |

#### 标签命名建议

| 文件类型 | 推荐标签 |
|---------|---------|
| 职业发展类 | 职业发展、面试、求职、工作经历、项目经验 |
| 认知成长类 | 认知升级、思维模式、牛马思维、做人思维、案例对比 |
| 家庭关系类 | 家庭、婆媳、领导力、代际关系、系统思维 |
| 健康管理类 | 健康、治疗记录、用药跟踪、医疗、干扰素 |
| 面试记录类 | 面试、求职、公司背景、岗位要求、面试总结 |

## 工作流程

### 1. 新增内容时的决策流程

**第一步：分析内容角度**
- 这段内容属于什么角度？（思维能力/案例/职场经历/战略/面试/家庭/健康/合同）
- 是否涉及多个角度？

**第二步：查阅 AGENTS.md 中的"知识库索引"表**
- 根据角度找到对应的目标文件
- 如涉及多个角度，确定需要更新的多个目标文件
- 默认复用现有文件，不新建文件

**第三步：读取目标文件**
- 读取现有文件内容，了解已有结构和内容
- 确定新内容应该插入的位置

**第四步：更新文件**
- 将内容按对应角度追加到目标文件
- 更新 `last_updated` 日期

**第五步：同步 AGENTS.md**
- 如有必要，更新"知识库索引"表的描述

### 2. 知识库文件操作

执行用户请求的增删改查操作：

- **创建 (Create)**: 在 `knowledge/` 目录下新建知识库文件，必须添加 YAML 文件头
  - **仅在用户明确要求，或现有文件无法容纳新内容角度时才创建**
- **读取 (Read)**: 读取指定知识库文件内容
- **更新 (Update)**: 修改现有知识库文件，同步更新 `last_updated` 日期
- **删除 (Delete)**: 删除知识库文件

### 3. 自动同步 AGENTS.md

每次操作完成后，必须更新 `AGENTS.md` 中的以下内容：

#### 3.1 更新"Skill 与知识库关系表"

如果新增/删除/重命名了知识库文件，且该文件被某个 Skill 使用，需要更新关系表中的对应行。

#### 3.2 更新"知识库索引"表

**位置**: `AGENTS.md` 中的"知识库索引"章节

**表格格式**:

```markdown
| 文件名 | 内容摘要 | 关键词/主题 |
|--------|---------|------------|
| `filename.md` | 简要描述文件内容 | 关键词1、关键词2、关键词3 |
```

**更新步骤**:

1. **读取当前 `knowledge/` 目录下的所有文件**
2. **读取每个文件的 YAML 文件头**，提取 `description` 和 `tags`
3. **更新索引表**:
   - 按字母顺序排列文件
   - 文件名列：使用代码格式 `` `filename.md` ``
   - 内容摘要列：使用文件头中的 `description`
   - 关键词列：使用文件头中的 `tags`，用顿号分隔
4. **写回 `AGENTS.md`**

## 使用示例

### 示例1：新增思维能力相关内容

```
用户：我最近在思考，我发现我有一种能力，能在压力下快速学习

执行：
1. 分析角度：个人特质与思维能力成长
2. 查阅 AGENTS.md 的"知识库索引"表 → 对应 `cognitive-growth.md`
3. 读取 `cognitive-growth.md` 了解现有结构
4. 在"我当下的特质和能力"章节追加新内容
5. 更新文件头部的 `last_updated` 日期
```

### 示例2：新增具体案例

```
用户：我今天遇到一个事，我发现我在用牛马思维处理问题...

执行：
1. 分析角度：做人vs牛马案例
2. 查阅 AGENTS.md 的"知识库索引"表 → 对应 `cognitive-cases.md`
3. 读取 `cognitive-cases.md` 了解案例编号和格式
4. 按格式追加新案例
5. 更新 `last_updated` 日期
```

### 示例3：一次对话涉及多个角度

```
用户：我今天面试了一家公司，面试过程中我发现自己还在用牛马思维...

执行：
1. 分析涉及的角度：
   - 面试详情
   - 思维反思
2. 查阅 AGENTS.md 的"知识库索引"表：
   - 面试详情 → `interview.md`
   - 思维反思 → `cognitive-cases.md`
3. 分别读取两个目标文件
4. 将面试详情记录到 `interview.md`
5. 将思维反思记录到 `cognitive-cases.md`
6. 分别更新两个文件的 `last_updated` 日期
```

### 示例4：新增工作经历

```
用户：我刚完成了一个新项目，想记录一下

执行：
1. 分析角度：职场经历与项目
2. 查阅 AGENTS.md 的"知识库索引"表 → 对应 `resume&projects.md`
3. 读取文件，找到工作经历部分
4. 在对应公司下追加项目经验，或新增公司章节
5. 更新 `last_updated` 日期
```

### 示例5：删除文件

```
用户：删除知识库中的 old-notes.md

执行：
1. 删除 knowledge/old-notes.md
2. 从 AGENTS.md 的"知识库索引"表中移除对应行
3. 检查"Skill 与知识库关系表"，如有引用则更新
```

## 注意事项

1. **优先复用现有文件**：新增内容时，默认先查阅 AGENTS.md 的"知识库索引"表，找到对应角度的现有文件进行更新；仅当用户明确要求或现有文件无法容纳时才新建文件
2. **必须包含文件头**：所有知识库文件必须包含 YAML 文件头
3. **必须同步更新**：任何对 `knowledge/` 目录的操作都必须同步更新 `AGENTS.md`
4. **保持格式一致**：使用统一的 YAML 格式和 Markdown 表格格式
5. **控制内容长度**：追加内容时应简洁精炼，参考现有文档风格，避免新章节长度超过之前2个章节的总和。如需记录大量内容，应：
   - 先提炼核心要点
   - 使用表格、列表等结构化方式呈现
6. **日期更新**：修改文件时，同步更新 `last_updated` 日期
7. **标签规范**：使用统一的标签命名，便于检索和分类
