# Knowledge Manager

## 职责

管理 `knowledge/` 目录下的知识库文件，执行增删改查操作，并在每次操作后自动同步更新 `agent.md` 文件中的相关内容。

## 知识库文件规范

### 文件位置

所有知识库文件必须存放在 `knowledge/` 目录下。

### 文件头格式（YAML Front Matter）

每个知识库文件必须包含统一的 YAML 文件头：

```yaml
---
title: "文件标题"
tags: ["标签1", "标签2", "标签3"]
category: "分类名称"
description: "文件内容简要描述"
last_updated: "YYYY-MM-DD"
---
```

#### 字段说明

| 字段 | 必填 | 说明 |
|------|------|------|
| `title` | 是 | 文件标题，简洁明了 |
| `tags` | 是 | 关键词标签，3-8个，用于检索和分类 |
| `category` | 是 | 所属分类，如：职业发展、健康管理、个人成长等 |
| `description` | 是 | 内容摘要，50字以内 |
| `last_updated` | 是 | 最后更新日期，格式：YYYY-MM-DD |

#### 标签命名建议

| 文件类型 | 推荐标签 |
|---------|---------|
| 职业发展类 | 职业发展、面试、求职、工作经历、项目经验 |
| 认知成长类 | 认知升级、思维模式、牛马思维、做人思维、案例对比 |
| 家庭关系类 | 家庭、婆媳、领导力、代际关系、系统思维 |
| 健康管理类 | 健康、治疗记录、用药跟踪、医疗、干扰素 |
| 面试记录类 | 面试、求职、公司背景、岗位要求、面试总结 |

## 工作流程

### 1. 知识库文件操作

执行用户请求的增删改查操作：

- **创建 (Create)**: 在 `knowledge/` 目录下新建知识库文件，必须添加 YAML 文件头
- **读取 (Read)**: 读取指定知识库文件内容
- **更新 (Update)**: 修改现有知识库文件，同步更新 `last_updated` 日期
- **删除 (Delete)**: 删除知识库文件

### 2. 自动同步 agent.md

每次操作完成后，必须更新 `agent.md` 中的以下内容：

#### 2.1 更新"Skill 与知识库关系表"

如果新增/删除/重命名了知识库文件，且该文件被某个 Skill 使用，需要更新关系表中的对应行。

#### 2.2 更新"知识库索引"表

**位置**: `agent.md` 中的"知识库索引"章节

**表格格式**:

```markdown
| 文件名 | 内容摘要 | 关键词/主题 |
|--------|---------|------------|
| `filename.md` | 简要描述文件内容 | 关键词1、关键词2、关键词3 |
```

**更新步骤**:

1. **读取当前 `knowledge/` 目录下的所有文件**
2. **读取每个文件的 YAML 文件头**，提取 `description` 和 `tags`
3. **更新索引表**:
   - 按字母顺序排列文件
   - 文件名列：使用代码格式 `` `filename.md` ``
   - 内容摘要列：使用文件头中的 `description`
   - 关键词列：使用文件头中的 `tags`，用顿号分隔
4. **写回 `agent.md`**

## 文件描述规则

根据文件名和内容自动推断描述：

| 文件名模式 | 默认描述 |
|-----------|---------|
| `resume*`, `cv*` | 简历与项目经验 |
| `cognitive*` | 认知发展与思维模式相关内容 |
| `family*` | 家庭系统分析与关系处理 |
| `health*`, `medical*`, `treatment*` | 健康数据与治疗记录 |
| `interview*` | 面试记录与总结 |
| `career*` | 职业发展与职场策略 |
| `project*` | 项目经验与案例 |
| 其他 | 根据文件内容前3行推断 |

## 使用示例

### 创建新文件

```
用户：在知识库中创建一个关于时间管理的文件

执行：
1. 创建 knowledge/time-management.md，添加 YAML 文件头
2. 更新 agent.md 中的"知识库索引"表，添加新行
```

### 删除文件

```
用户：删除知识库中的 old-notes.md

执行：
1. 删除 knowledge/old-notes.md
2. 从 agent.md 的"知识库索引"表中移除对应行
3. 检查"Skill 与知识库关系表"，如有引用则更新
```

### 重命名文件

```
用户：将 cognitive-growth.md 重命名为 mind-development.md

执行：
1. 重命名文件
2. 更新 agent.md"知识库索引"表中的文件名
3. 更新"Skill 与知识库关系表"中的引用（如有）
```

### 为现有文件添加/更新文件头

```
用户：给 interview.md 加上文件头

执行：
1. 读取文件内容，理解主题
2. 在文件开头添加 YAML 文件头
3. 更新 agent.md"知识库索引"表中的描述和关键词
```

## 注意事项

1. **必须包含文件头**：所有知识库文件必须包含 YAML 文件头
2. **必须同步更新**：任何对 `knowledge/` 目录的操作都必须同步更新 `agent.md`
3. **保持格式一致**：使用统一的 YAML 格式和 Markdown 表格格式
4. **日期更新**：修改文件时，同步更新 `last_updated` 日期
5. **标签规范**：使用统一的标签命名，便于检索和分类
