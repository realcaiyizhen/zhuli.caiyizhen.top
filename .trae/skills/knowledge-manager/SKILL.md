# Knowledge Manager

## 核心职责

管理知识库文件的增删改查，确保内容**精简、合并、不冗余**。

## 黄金法则（必须遵守）

> **追加 = 错误，合并 = 正确**
> 
> **每次写入前必须问自己：我是在追加新章节，还是在合并到现有章节？**
> 
> **如果答案是"追加"，停止，重新思考如何合并。**

## 用户指令优先级

| 用户指令 | 执行动作 |
|---------|---------|
| "把对话内容更新到知识库" | 分析对话 → 提取核心要点 → **合并到现有章节** → 控制长度 |
| "合并到XXX章节" | 将新内容**整合**到指定章节，替换或整合原有内容 |
| "控制长度" | 严格压缩内容，只保留核心洞察和关键结论 |
| "新建知识库文件" | 按规范创建新文件，文件名翻译成英文 |

## 追加内容的强制规则（违反任何一条都是错误）

### 规则1：必须先读取目标文件
**禁止**在未读取现有内容的情况下直接追加。

### 规则2：必须合并而非追加（最关键）
- ❌ **禁止**在文件末尾新建章节
- ❌ **禁止**创建"最新发现"、"补充"、"更新"等章节
- ✅ **必须**找到与新内容相关的**现有章节**
- ✅ **必须**将新内容**整合**进现有章节
- ✅ **必须**删除重复表述，保留核心差异

**合并检查清单（写入前必须确认）**：
- [ ] 我没有创建新章节
- [ ] 我把新内容融入了现有段落
- [ ] 我删除了重复内容
- [ ] 总长度没有显著增加

### 规则3：必须控制长度
- 单条追加内容**不超过5行**
- 如需表格，**不超过4行**
- 优先使用**一句话总结**替代段落
- 如果合并后章节过长，必须精简现有内容

### 规则4：必须用户确认
执行写入前，先向用户展示：
- 目标文件
- 合并位置（具体章节）
- 最终内容（精简后）
- 确认后再写入

## 工作流程

### 场景1：更新现有知识库文件

```
用户：把对话内容更新到知识库
    ↓
1. 分析内容角度（认知/职业/家庭/健康等）
    ↓
2. 查阅 AGENTS.md → 确定目标文件
    ↓
3. 读取目标文件 → 找到相关章节
    ↓
4. 【关键】合并内容：
   - 提取新内容核心要点（1-3点）
   - 找到现有相关章节
   - **整合进现有段落，删除重复**
   - **禁止新建章节**
   - 控制长度（≤5行或1个表格）
    ↓
5. 向用户展示合并方案
    ↓
6. 用户确认后写入
    ↓
7. 更新 last_updated
```

### 场景2：新建知识库文件（**必须执行以下全部步骤**）

```
用户：新建知识库文件 / 保存到知识库
    ↓
1. 确定文件名（英文，小写，连字符连接）
    ↓
2. 创建文件，写入完整内容 + YAML front matter
    ↓
3. 【必须】更新 AGENTS.md 知识库索引表格：
   - 读取 AGENTS.md 找到"知识库索引"表格
   - 添加新文件条目：文件名、内容摘要、关键词
   - 保持表格按字母顺序或逻辑顺序排列
    ↓
4. 【必须】检查 Skill-知识库关系表：
   - 如果新文件被某个 Skill 使用，更新对应表格
   - 如果未被使用，无需更新
```

**⚠️ 新建文件后忘记更新 AGENTS.md 是常见错误，必须避免！**

## 内容合并示例

### 错误做法（直接追加）❌
```markdown
## 原有章节
内容A...

## 新章节（2026-04-08）  ← 错误！重复建章节
内容B...

## 最新发现  ← 错误！又建章节
内容C...
```

### 正确做法（合并整合）✅
```markdown
## 原有章节
内容A... + 内容B核心要点（一句话）+ 内容C核心要点（一句话）
```

## 常见错误模式（必须避免）

| 错误模式 | 正确做法 |
|---------|---------|
| 创建"最新发现"章节 | 合并到"当前处境"或相关章节 |
| 创建"补充"章节 | 整合到原有段落 |
| 保留重复描述 | 删除重复，只保留差异点 |
| 长篇累牍追加 | 压缩到5行以内 |
| 多个表格 | 合并成一个精简表格 |

## 长度控制技巧

| 原内容 | 压缩后 |
|--------|--------|
| 3个段落描述 | 1句话总结 |
| 详细表格（6行+） | 精简表格（≤4行）或删除 |
| 多个案例 | 保留1个最典型案例 |
| 重复强调 | 删除，只保留一次 |

## 文件规范

### 新建文件命名规则
- 用户用中文描述文件名时，**翻译成英文**
- 使用小写字母，单词间用连字符 `-` 连接
- 示例：
  - "我的认知成长" → `my-cognitive-growth.md`
  - "面试记录" → `interview-records.md`
  - "家庭系统" → `family-system.md`

### YAML 文件头
```yaml
---
title: "标题"
tags: ["标签1", "标签2"]
category: "分类"
description: "50字内描述"
last_updated: "YYYY-MM-DD"
---
```

### 更新时必须
1. 更新 `last_updated` 日期
2. 同步更新 AGENTS.md 索引（如有必要）

## 禁止事项

- ❌ 未读取文件直接追加
- ❌ **新建章节而不合并（最常见错误）**
- ❌ 内容超过5行不压缩
- ❌ 未经确认直接写入
- ❌ 保留重复表述
- ❌ 创建"最新"、"补充"、"更新"等章节
