# 何时使用

当用户出现以下情况时调用此技能：

- 讨论职场思维、职业发展或职业转型
- 探索职业机会或职业风险管理
- 任何面试或者公司行业分析的相关问题
- 面试场合，对面试官进行演示，此时面试官会问我各种职业问题，你可以代替我回答

## 面试记录管理

### interview.md 文件用途

`knowledge/interview.md` 文件用于记录每次面试的详细信息，包括公司背景、岗位要求、面试准备、面试过程。该文件会不断追加新的面试记录，用于总结经验、优化面试策略。

### 文件格式规范

每次面试记录应包含以下结构：

```markdown
## [公司名称] [岗位名称]

### 基本信息
- **公司名称**：xxx
- **面试岗位**：xxx
- **面试时间**：xxxx年xx月xx日xx点
- **面试官**：xxx

### 公司背景
- **公司简介**：xxx
- **业务范围**：xxx
- **战略方向**：xxx

### 岗位要求
- **岗位职责**：xxx
- **任职要求**：xxx

### 面试准备
- **能力差距分析**：xxx
- **管理经验重构**：xxx
- **探索型团队考核方案**：xxx

### 面试记录
- **面试时间**：xxx
- **面试官**：xxx
- **面试问题**：xxx
- **面试反馈**：xxx
- **后续安排**：xxx
```

### 如何追加新的面试记录

1. **在文件末尾添加新的二级标题**：
   ```markdown
   ## [公司名称] [岗位名称]
   ```

2. **按上述格式填写完整信息**：
   - 基本信息（公司名称、面试岗位、面试时间、面试官）
   - 公司背景（公司简介、业务范围、战略方向）
   - 岗位要求（岗位职责、任职要求）
   - 面试准备（能力差距分析、管理经验重构、探索型团队考核方案等）
   - 面试记录（面试时间、面试官、面试问题、面试反馈、后续安排）

3. **更新文件末尾的最后更新时间**：
   ```markdown
   ---
   *最后更新：2026-03-25*
   ```

### 注意事项
- 如实记录，不添加假设内容，无需你给出任何建议。
